hugo和Git命令

八月 16, 2019 发布在 技术小白, hugo和git

　　记录一下，忙活了两天，这才算自己的第一份博客吧。之前没有接触过hugo,也没有用过git命令，所以着手搭建属于自己的一个博客的过程中可谓坎坷，总是会出现一些莫名其妙的ERROR，简直让人抓狂。最后还是耐着性子和请教大佬慢慢的解决· · ·

目录

    • 下载Hugo和Git
    • 下载主题并设置主题
    • 写一篇文章

　　这一篇博客就简单介绍一下自己从开始有这个念头到成功搭建所用的命令和方法，最后再汇总一下常用的git命令，也算是方便以后自己查阅。

下载Hugo和Git

　　1.作为windows用户，首先在hugo的github官网：https://github.com/gohugoio/hugo/releases 上下载Windows版本的的hugo安装包，
　　下载好之后，解压缩（可以自己选择文件路径，不要带有中文），之后便将hugo添加到windows环境变量中，添加好之后使用win+R打开运行窗口输出cmd,打开命令行，输入:hugo versin,提示版本号如: Hugo Static Site Generator v0.57.0-9B00E647 windows/amd64 BuildDate: 2019-08-14T08:05:40Z
说明hugo已经配置好了。Git的下载安装直接进入Git官网点击右侧download，然后会自动弹出下载框，按照提示安装即可。 ##生成hugo博客 　　2.配置好之后首先要用hugo生成一个blog根目录，在命令行窗口下输入命令即可。hugo new site myblog

下载主题并设置主题

    3.现在就需要下载一个主题，可以在hugo主题官网上挑选喜欢的主题，挑选完主题之后，在命令行根目录下（也就是myblog目录），输入命令(这里以我的主题为例)，等待下载好之后便成功设置好主题。
git clone https://github.com/kakawait/hugo-tranquilpeak-theme.git

写一篇文章

　　4.首先，在根目录下使用命令：hugo new post/firstbolg.md，这样就可以在myblog的content文件下生成一个post文件夹，以后写文章都是存在这个文件里，里面就有我们刚生成的firstblog文件。接着我们可以使用VScode编译器打开firstblog文件，便可以在里面编辑我们的文章。

本地启动博客

　　5.接着就可以在本地启动博客，查看自己之前下载的主题和自己写的文章，在根目录下使用命令：huto server -t huto-tranquilpeak-theme --buildDrafts，之后便会生成一个本地目录，在浏览器中输入这个地址便可以在本地启动博客。

部署到远程服务器github

　　6.首先创建一个github账号，然后点击头像，然后点击new projiects,然后在Repositories中点击右侧New,然后起一个仓库名，XXXX就是前面Owner中的名字。起好之后使用命令部署(XXXX就是之前起的名字)hugo --themes=hugo-tranquilpeak-theme --baseUrl="https//github.com/XXXX/XXXX.github.io/" --buildDrafts执行完命令之后就会在根目录下出现一个public文件夹，使用cd public命令进入public文件夹，然后使用命令git init创建本地仓库，然后再使用命令git add .把public中所有文件加进去，接着写一个提交信息git commit -m "XXXX"(XXXX为自己随便写)，然后跟远端仓库关联，使用命令git remote add origin https://github.com/XXXX/XXXX.github.io.git(XXXX和之前一样)，最后使用命令把文件push到远程仓库git push -u origin master这样便成功把博客部署到远程git仓库，之后便可在浏览器输入XXXX.github.io访问自己的博客。

总结与git命令

　　7.总的来说，说的还是比较概括，其中注册github账号后创建SSH Key和和填写公钥的过程也没有讲，其实只有动手之后才发现自己的不足，之前完全没有了解过git命令，所以实际操作起来后遇到了很多问题，在此，总结一下自己用到的各种命令： cd: 改变目录 #cd myblog 就是改变目录进入到myblog
cd .. 回退到上一个目录 #cd和..中间有空格
mkdir myblog 创建一个myblog文件夹
vi myblog 编辑文件myblog :wq 推出编辑，返回命令行
git init 创建本地仓库
git add . 添加所有文件到本地仓库
git commit -m"xxx" 提交一个仓库信息
git remote add origin https://github.com/xxx/xxx.github.io/.git 跟远程仓库关联
git pull origin master 从远程拉取master分支与本地合并 git push -u origin master 从本地提交到远程master分支
